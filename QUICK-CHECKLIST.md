## 🚨 긴급 상황

### localStorage 가득 참 (QuotaExceededError)

```bash
1. Export로 백업 (필수!)
2. 브라우저 콘솔(F12) 열기
3. 다음 코드 실행:
   localStorage.clear()
   location.reload()
4. Import로 백업 복원
5. API 키 다시 입력
```

### 노드가 삭제 안됨

```bash
1. Export로 백업
2. localStorage 정리 (위 방법)
3. 새로고침
4. Import로 복원
```

### 앱이 느려짐

```bash
1. 노드 개수 확인 (100개 이상이면 위험!)
2. 사용하지 않는 노드 삭제
3. 워크플로우 분리 (Export → 새 탭 → Import)
4. 브라우저 재시작
```

## 📊 저장공간 상태 해석

### localStorage

| 사용률 | 상태 | 행동 |
|--------|------|------|
| 0-70% | ✅ 안전 | 계속 작업 |
| 71-90% | ⚠️ 주의 | Export로 백업 준비 |
| 91-100% | 🚨 위험 | 즉시 정리 필요! |

### 노드 개수

| 개수 | 상태 | 행동 |
|------|------|------|
| 0-50 | ✅ 최적 | 문제 없음 |
| 51-100 | ⚠️ 주의 | 정리 권장 |
| 101+ | 🚨 위험 | 워크플로우 분리 필수 |

### IndexedDB

| 이미지+비디오 | 상태 | 행동 |
|--------------|------|------|
| 0-50개 | ✅ 정상 | 문제 없음 |
| 51-100개 | ⚠️ 주의 | 자동 정리 확인 |
| 101개+ | 💡 팁 | 30일 후 자동 삭제됨 |

## 🎯 골든 룰 (절대 지켜야 할 3가지)

### 1️⃣ Export 없이 정리 금지
```
❌ localStorage 정리 → Export
✅ Export → localStorage 정리
```

### 2️⃣ 100개 노드 넘기지 말기
```
50개 넘으면 → 정리 시작
100개 가까우면 → 워크플로우 분리
```

### 3️⃣ 주기적 백업
```
30분마다 → Save
1시간마다 → Export
작업 끝날 때 → Export (필수!)
```
